Это тестовый документ для проверки загрузки. Он содержит несколько абзацев текста, которые должны быть разделены на отдельные чанки при обработке. Каждый чанк должен получить свой собственный эмбеддинг. Давайте посмотрим, сколько вызовов к API эмбеддингов будет сделано.

Второй абзац содержит дополнительную информацию о тестировании системы. Здесь мы проверяем работу кэша и эффективность обработки больших документов. Система должна корректно разделять текст на фрагменты и избегать дублирования вычислений.

Третий абзац нужен для того, чтобы создать достаточно большой документ. В реальных условиях документы могут быть намного больше, и важно убедиться, что система справляется с ними эффективно. Повторная загрузка того же документа должна использовать кэш и не делать лишних вызовов к OpenAI.
