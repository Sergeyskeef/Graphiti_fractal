Для заливки в графы - подробный анализ производительности

Этот файл содержит текст, который при загрузке в систему будет разбит на несколько чанков. 
Каждый чанк проходит через процессинг: чанкирование, эмбеддинги, Graphiti обработку.

Анализ производительности показывает следующие узкие места:

1. Эмбеддинги - вызовы к OpenAI API для получения векторных представлений текста
2. Graphiti.add_episode - сложная обработка с извлечением сущностей, связей, фактов
3. Neo4j операции - запись в графовую базу данных

Типичные проблемы производительности:

- Rate limits от провайдера эмбеддингов (429 Too Many Requests)
- Последовательная обработка вместо параллельной
- Слишком детальная обработка каждого чанка (много LLM вызовов)

Решения для оптимизации:

1. Увеличить лимиты параллелизма в Graphiti (max_coroutines)
2. Использовать более эффективные модели эмбеддингов
3. Отключить ненужные этапы обработки для bulk загрузки
4. Оптимизировать Neo4j индексы и запросы

Текущие настройки системы:
- max_coroutines: по умолчанию (вероятно 10-20)
- Embedding model: text-embedding-3-small (1536 измерений)
- Graphiti pipeline: полная обработка (entities, edges, facts)
- Neo4j: стандартная конфигурация

Измерения показывают, что на один чанк уходит:
- 2-5 вызовов эмбеддингов (по 1-3 секунды каждый)
- 5-15 секунд на Graphiti обработку
- 1-2 секунды на Neo4j операции

Итого: 10-25 секунд на чанк, что для 11 чанков дает 2-5 минут общей обработки.
